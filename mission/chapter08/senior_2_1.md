# Rest Docs

### 1️⃣ Rest Docs가 무엇인지 알아보기

**테스트 코드를 기반으로 API 문서를 자동 생성하는 도구**이다.

대표적으로 Spring REST Docs를 의미하며, Spring MVC Test / RestAssured 테스트 결과를 바탕으로

**“실제로 검증된 API”** 에 대해서만 문서를 만든다는 특징이 있다.

 

| 개념 | 설명 |
| --- | --- |
| 테스트 기반 | MockMvc / RestAssured 테스트가 통과해야 문서를 생성 |
| 스니펫(snippet) | 요청/응답/헤더/파라미터 구조 등을 조각 파일로 생성 |
| Asciidoctor 통합 | 최종적인 HTML 문서로 변환하여 배포 가능 |
| 정확성 보장 | 문서 = 테스트 결과 → API 변경 시 테스트가 실패해 문서 최신성 보장 |

### 왜 만들어졌는가?

기존 문서화 방식은 **코드와 문서가 계속 불일치**하는 문제가 있었다

그래서 API 문서를 **테스트에 의해 자동으로 생성**해 “실제 동작”과 “문서”를 일치시키려는 목적을 가지고 만들어졌다. 즉, 문서가 코드의 실제 동작을 증명해야 한다는 테스트 기반 문서화(REST Docs)의 철학을 따른다!
